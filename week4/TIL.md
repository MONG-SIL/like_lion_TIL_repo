# Week4 TIL

### 1. 오늘 배운 내용
- 정적 웹페이지 환경에서 데이터(배열/객체)와 화면(DOM)을 항상 같은 상태로 유지하는 방법
- 페이지 로드 시 HTML에 이미 있는 카드/상세 영역을 읽어 lions 배열로 초기화하는 패턴
- 외부 API(fetch)로 받은 데이터를 화면에서 쓰는 형태로 객체 변환(매핑)하는 방법
- 비동기 상태 UI(준비/로딩/성공/실패/재시도) 만들기와 요청 중 버튼 비활성화로 중복 요청 방지
- 보기 옵션(필터/정렬/검색)을 바꿔도 새로고침 없이 즉시 화면이 갱신되게 구성하기

### 2. 핵심 정리 (내 언어로)
- 초기 DOM을 읽어서 배열 초기화
  - summary-card, detail-card에 들어있는 텍스트(이름/파트/기술/연락처/한마디 등)를 파싱해서 lions 배열에 넣는다.
  - 초기화가 끝나면 총 n명은 배열 길이를 기준으로 갱신하면 화면과 데이터가 어긋나지 않는다.

- 단일 기준은 데이터, 화면은 그 결과
  - 추가할 때 DOM을 append처럼 DOM 조작이 분산되면 동기화 실수가 생기기 쉽다.
  - 그래서 데이터가 바뀔 때마다 render 함수로 요약/상세를 통째로 다시 그리는 방식이 안정적이다.

- 보기 옵션은 렌더링 전에 계산한다
  - getVisibleLions에서 필터(파트) -> 검색(이름) -> 정렬을 적용한 뒤, 그 결과만 렌더링한다.
  - 조건 때문에 0개가 되면 Empty State 문구를 보여주면 사용자가 왜 안 보이는지 바로 이해한다.

- fetch는 요청 흐름까지 포함해서 구현해야 한다

- 전체 새로고침 규칙
  - 내 카드를 제외한 명단을 새 데이터로 교체한다.
  - 버튼 클릭 시점의 총 인원 수는 유지되어야 한다.
