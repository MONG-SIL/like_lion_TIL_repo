# 📘 Week3 TIL

### 1. 오늘 배운 내용
- 정적 웹페이지에서 데이터(배열/객체) ↔ 화면(DOM)을 동기화하는 방법
- DOMContentLoaded 이후에만 JS 실행해서 DOM 안전하게 접근하기
- 폼 입력값 검증 + 입력 아래 경고 메시지 출력 구현
- 추가/삭제 시 요약 카드(그리드) / 상세 카드(1열 목록) / 총 인원 표시를 함께 갱신하기

### 2. 핵심 정리 (내 언어로)
- 초기 DOM을 읽어서 배열 초기화
  - HTML에 이미 들어있는 카드들을 querySelectorAll로 찾아서 필요한 값을 읽어 배열로 만든다.
  - 초기화가 끝나면 총 n명을 배열 길이로 갱신하면 화면과 데이터가 같은 상태로 시작할 수 있다.

- 배열이 바뀌면 DOM도 같이 바뀌게 만들기 (명단 데이터와 화면 동기화)
  - 추가: 배열에 push하고, DOM은 createElement로 만든 뒤 append로 맨 끝에 추가한다.
  - 삭제: 배열에서 pop하고, DOM도 lastElementChild.remove()로 마지막 카드들을 같이 제거한다.


- 폼 검증(Validation) + 에러 메시지 위치
  - 제출 시 FormData로 입력값을 읽고, 비어 있으면 추가를 막는다.
  - 경고는 input 아래에 있는 p.field__error에 텍스트를 넣어 보여준다.
  - 관심 기술은 쉼표로 split해서 배열로 만들고, 첫 번째 항목은 요약 카드 배지, 전체 목록은 상세 카드의 불렛 리스트로 표시.

- 이벤트 등록 방식
  - HTML에 onclick 같은 인라인 이벤트 없이, JS에서 addEventListener로만 등록한다.
  - 토글/추가/취소/삭제 같은 동작이 명확해져서 관리하기 편하다.

### 4. 느낀 점
- 화면을 그리는 코드보다 더 중요한 건 데이터를 단일 기준으로 두고, 화면은 그 결과로 유지하는 구조라는 걸 체감했다.
- 초기 HTML을 그대로 두고도 JS에서 읽어서 상태로 만드는 방식이 실전에서도 유용할 것 같다.
